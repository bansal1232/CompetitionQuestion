Frequency Threshold

You need to implement a data structure which can handle the following operations - 

  • Insert an integer X in the data structure
  • Delete an integer X from the data structure if it exists or else skip the query.
  • Print the sum of all integers whose frequency is greater than or equal to a particular threshold after every operation of the above two types

The data structure is initially empty. The insert query is denoted by the character i followed by an integer, and the delete query is denoted by the character d followed by an integer.

Input
The first line contains an integer Q as input and an integer T that denotes the total numbers of operations you need to perform and the threshold value of frequency respectively. 

Output
After each query, you need to print the sum of all integers whose frequency is greater than the threshold T .

Constraints
1≤Q≤105
1≤X≤109
1≤T≤109

Sample Input
5 2
i 4
i 4
i 3
i 3
d 4
Sample Output
0
4
4
7
3
Explanation

Here is the status of the data structure after every operation in the sample test case:

[4][4,4][4,4,3][4,4,3,3][4,3,3]
For every status, you can calculate the sum of integers with frequency ≥2 as it is the threshold limit.

Array Rebuild

You are given an array A of integers, which is initially empty. Now some new integers need to be added to this array. When you add an integer X , you find the value which is just smaller than it (denoted by a) and a value which is just larger than it (denoted by b) among the elements added till then in the array. If one of the values is missing i.e. a or b then it becomes 0 . The cost of inserting the new element is given by a×b .

Input
The first line contains an integer N that denotes the total number of integers that need to be inserted in the array. The next line contains N space separated integers.

Output
In the output, you need to print the total cost modulo 109+7 after all the integers are inserted into the array.

Constraints
1≤N≤105
1≤X≤109where X is the value that is inserted into the array.

Sample Input
3
9 1 2
Sample Output
9
Explanation

Initially, the array is empty so when 9 is added then the cost involved will be 0∗0=0.
Now 9 exists in the array and 1 is added, the cost involved is 0∗9=0 as there is no integer which is less than 1.
Now you insert 2, the cost involved is 1∗9=9 because 1 is just smaller than 2 and 9 is just larger than 2 which is present in the array.
Thus the total cost involved is 0+0+9=9.

String Division

You are given two strings of the same length. In one operation, you can pick one position from the first string and one from the second string and swap their values. Using these operations, you need to make the first string lexicographically smaller than the second string. If it is not possible then print −1 .

Input
The first line contains an integer t as input that denotes the total number of test cases. In each test case, there are two strings each on a new line.

Output
In the output, you need to print the minimum operations required.

Constraints
1≤t≤10
1≤|S|≤105 where |S| is the length of the string.
 

Sample Input
2
bnt
ait
apple
appls
Sample Output
1
0
Explanation

In the first test case, you can swap the first character in both the strings so the first string becomes ant and second string becomes bit, thus the first string becomes lexicographically smaller in one swap.
In the second test case, apple is already lexicographically smaller than the string appls, so the total swap required are 0.
